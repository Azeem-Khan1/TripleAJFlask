The TripleAJ Backend

How This Works

  • This is merely a homepage for our backend database.
  • Our arcade is dependent on a database, which is deployed on this flask framework.
  • We've used Github to track progress on our arcade and make issues every so often.
  • Here is a link to our scrum board.

What does the Backend Process look like?

  • The database stores a player's username (uid), encrypted password, token count, and full name.
  • SQL and SQLAlchemy are used to keep all of the data persistent... meaning that if something were to go wrong with the server or host, then the data would not be lost.
  • The frontend and backend are connected through the use of Javascript's Fetch API, which allows the frontend to call or fetch a backend Flask endpoint.
  • This information that is sent to the backend is considered a request (ie GET, POST). A Python function interprets the request, typically into a setter or getter against an SQL row and provides a response. Examples are user “signs up”, user “login” or game "spending/earning tokens" updates.
  • Our API can be found on the navigation bar above.
